# 🌦️ MRL Forecast Pro: Метеорологическая ИИ-платформа

## Статус реализации

Проект находится в активной модернизации. Operational-контур NOAA NEXRAD теперь
использует единый versioned pipeline `radar-grid-v1`, явный контроль источника и
quality gate модели. Синтетические данные доступны только через отдельный
`DEMO`-режим и не должны использоваться для production-обучения.

Основной план и ограничения: [`plan/README.md`](plan/README.md).

Этот проект представляет собой профессиональную систему краткосрочного прогнозирования (наукастинга) метеорологических радиолокационных данных с использованием глубокого обучения (Convolutional LSTM). Платформа обеспечивает полный цикл работы с данными: от автоматического сбора архивов до обучения и применения ИИ-моделей.

## ✨ Ключевые возможности

- **🧠 Нейросетевой движок**: ConvLSTM с поддержкой композитного обучения на нескольких датасетах, temporal split и сравнением с persistence/advection baseline.
- **🔄 Полный жизненный цикл (Lifecycle Management)**: 
  - Автоматизированный трекингProvenance: история обучения модели (какие данные использовались).
  - Тонкая настройка валидации и заблаговременности (Lead Time).
- **📡 Интеграция прогнозов (Assimilation)**:
  - Экспорт ИИ-прогнозов в формат **NetCDF4** для интеграции с внешними системами (WRF, GIS).
- **📡 Универсальные Дата-Адаптеры**:
  - **NOAA FTP (Live)**: Оперативное скачивание данных NEXRAD Level III.
  - **Archive Explorer**: Массовая загрузка NOAA NEXRAD Level II из публичного AWS S3 бакета `unidata-nexrad-level2`.
  - **DEMO**: Явно маркированные синтетические кадры только для проверки UI.
- **🖥️ Professional Web UI (Pro Dashboard)**:
  - **Инференс**: Горячее переключение между моделями из реестра с мгновенным обновлением прогноза.
  - **Менеджер данных**: Умная группировка по станциям и датам, инвентаризация и управление загруженными архивами.
  - **Лаборатория моделей**: Запуск обучения на выбранных датасетах с мониторингом логов и метрик.
  - **Data Lifecycle**: Полная визуализация пути данных от сырого архива до обученной модели.

## 🚀 Быстрый старт (Quick Start)

### Для Linux / macOS:
1.  **Настройте окружение**: `bash scripts/setup.sh`
2.  **Запустите платформу**: `bash scripts/run_app.sh`
3.  **Проверьте окружение**: `bash scripts/doctor.sh`

### Для Windows 11:
1.  **Настройте окружение**: Запустите двойным кликом `scripts\setup.bat`
2.  **Запустите платформу**: Запустите двойным кликом `scripts\run_app.bat`

Откройте браузер: `http://localhost:5005`.

## 💻 Установка на Windows 11 (Подробно)

Для корректной работы системы на Windows 11 выполните следующие шаги:

### 1. Установка необходимого ПО
*   **Python 3.10+**: [Скачать с python.org](https://www.python.org/downloads/windows/). 
    *   *Важно*: При установке обязательно отметьте галочку **"Add Python to PATH"**.
*   **Git**: [Скачать с git-scm.com](https://git-scm.com/download/win).
*   **Microsoft C++ Build Tools**: [Скачать здесь](https://visualstudio.microsoft.com/visual-cpp-build-tools/).
    *   Выберите рабочую нагрузку *"Разработка классических приложений на C++"* (необходимо для компиляции некоторых библиотек, таких как `netCDF4` и `arm-pyart`).

### 2. Подготовка проекта
1.  Склонируйте репозиторий:
    ```bash
    git clone https://github.com/your-repo/mrl_forecast.git
    cd mrl_forecast
    ```
2.  Запустите файл автоматической настройки: `scripts\setup.bat`. 
    *   Скрипт создаст виртуальное окружение `venv`, обновит `pip` и установит все зависимости.
    *   Также будут созданы необходимые папки для данных и моделей.

### 3. Запуск приложения
Запустите `scripts\run_app.bat`. Консоль должна отобразить процесс загрузки модели и адрес сервера.

## 🌐 Управление через веб-интерфейс (Рекомендуется)

Платформа MRL Forecast Pro позволяет управлять всем циклом без использования терминала:

1.  **Загрузка**: В «Менеджере данных» выберите станцию и дату. Система создаст уникальную папку с метаданными.
2.  **Подготовка**: Нажмите «Подготовить» у загруженного архива. Это преобразует сырые данные в обучающую выборку.
3.  **Обучение**: В «Лаборатории моделей» выберите датасет, настройте гиперпараметры и запустите обучение. Модель автоматически сохранится в реестр.
4.  **Прогноз**: На вкладке «Прогноз» выберите вашу лучшую модель из выпадающего списка.

## 🧬 Работа через терминал (Terminal)

Для продвинутых пользователей доступны командные интерфейсы (Bash для Linux/macOS, Batch для Windows):

### Шаг 1: Скачивание архива
```bash
# Linux/macOS
bash scripts/download.sh KOKX 2024-05-20 100

# Windows
scripts\download.bat KOKX 2024-05-20 100
```
*Данные сохраняются в `data/raw/archive/<ID_СЕССИИ>`.*

### Шаг 2: Создание датасета
```bash
# Linux/macOS
bash scripts/prepare.sh 8 data/raw/archive/<ID_СЕССИИ>

# Windows
scripts\prepare.bat 8 data/raw/archive/<ID_СЕССИИ>
```

### Шаг 3: Обучение модели
```bash
# Linux/macOS
bash scripts/train.sh 20 4 1e-4 data/processed_archive/<ID_ДАТАСЕТА>

# Windows
scripts\train.bat 20 4 1e-4 data/processed_archive/<ID_ДАТАСЕТА>
```
*Чекпоинты и метаданные сохраняются в `models/registry/`.*

### Диагностика источника AWS

```bash
venv/bin/python scripts/check_aws_source.py --station KOKX --date 2024-05-20
```

Для загрузки и декодирования одного контрольного файла добавьте `--decode-one`.

### Аудит артефактов

```bash
venv/bin/python scripts/audit_baseline.py \
    --dataset-dir data/processed_archive/<ID_ДАТАСЕТА>
```

## Ограничения

- Модели и датасеты без `pipeline_version` считаются legacy.
- Модель со статусом `training`, `failed` или `rejected_quality_gate` нельзя
  выбрать для operational-инференса.
- Поддержка российских ДМРЛ остается отдельным контуром до получения реальных
  fixtures и подтверждения источника данных.
