# 🌦️ MRL Forecast: ИИ-Наукастинг Осадков

Этот проект представляет собой систему краткосрочного прогнозирования (наукастинга) метеорологических радиолокационных данных с использованием глубокого обучения (Convolutional LSTM). Система автоматически предсказывает перемещение и развитие зон осадков на ближайшее время.

## ✨ Ключевые возможности

- **🧠 Нейросетевой движок**: Реализация многослойной сети ConvLSTM на PyTorch для пространственно-временного прогнозирования.
- **📡 Универсальные Дата-Адаптеры**:
  - **NOAA FTP (Live)**: Автоматическое скачивание и декодирование сырых данных NEXRAD Level III.
  - **Локальные архивы**: Декодирование файлов формата **BUFR** (МРЛ/ДМРЛ) и NEXRAD Level II.
- **🗺️ Визуализация**: Рендеринг данных с наложением колец дальности и стандартизированной цветовой шкалой отражаемости (dBZ).
- **🖥️ Web UI**: Дашборд на Bootstrap 5 для мониторинга и прогноза в реальном времени.

## 🚀 Быстрый старт

1. **Клонируйте репозиторий и настройте окружение**:
   ```bash
   git clone <url-репозитория>
   cd mrl_forecast
   bash scripts/setup.sh
   ```

2. **Запустите приложение** (с использованием стандартной модели):
   ```bash
   bash scripts/run_app.sh
   ```
   Откройте браузер по адресу `http://localhost:5005`.

## 🌐 Управление через веб-интерфейс

Теперь вы можете управлять всем циклом проекта без использования терминала:

1.  **Загрузка данных**: Перейдите во вкладку «Загрузка данных», выберите метеостанцию и дату. Процесс скачивания будет отображаться в интерактивном окне логов.
2.  **Обучение**: Во вкладке «Обучение» настройте гиперпараметры и запустите процесс. Система автоматически подготовит датасет и начнет обучение, транслируя прогресс в браузер.
3.  **Применение**: После завершения обучения новая модель автоматически станет доступна для использования во вкладке «Прогноз».

## 🧬 Полный цикл ИИ: От данных до прогноза (через терминал)

Для обучения собственной модели на новых данных следуйте этой инструкции:

### Шаг 1: Скачивание архивных данных
Используйте скрипт `scripts/download.sh` для загрузки сырых данных NEXRAD Level II.
```bash
# Использование: bash scripts/download.sh [ID_СТАНЦИИ] [ДАТА] [КОЛ-ВО_ФАЙЛОВ]
bash scripts/download.sh KOKX 2024-05-20 100
```
- `STATION`: ID метеостанции (например, KOKX — Нью-Йорк).
- `DATE`: Дата в формате ГГГГ-ММ-ДД.
- `COUNT`: Количество файлов сканирования (рекомендуется от 50 до 200 для одного дня).

### Шаг 2: Подготовка обучающего датасета
Преобразуйте бинарные файлы радара в последовательности Numpy, готовые для обучения.
```bash
# Использование: bash scripts/prepare.sh [ДЛИНА_ПОСЛЕДОВАТЕЛЬНОСТИ]
bash scripts/prepare.sh 8
```
- `SEQ_LEN`: Общая длина последовательности (например, 8 означает 4 кадра истории + 4 кадра для прогноза).
- Скрипт считывает данные из `data/raw/archive` и сохраняет результат в `data/processed_archive`.

### Шаг 3: Обучение нейросети ConvLSTM
Запустите процесс обучения на подготовленных данных.
```bash
# Использование: bash scripts/train.sh [ЭПОХИ] [БАТЧ] [LEARNING_RATE]
bash scripts/train.sh 20 4 1e-4
```
- Модель будет сохранять лучший чекпоинт в `models/real_checkpoints/best_model.pt`.
- Логи обучения сохраняются в ту же папку.

### Шаг 4: Запуск системы с новой моделью
После обучения вы можете запустить веб-приложение, которое будет использовать вашу обученную модель:
```bash
bash scripts/run_app.sh 5005 models/real_checkpoints/best_model.pt
```

## ⚙️ Настройка параметров
Если вы хотите изменить параметры сетки (разрешение, дальность), отредактируйте переменные в `src/make_dataset.py` (функция `process_archive_directory`). По умолчанию используется сетка 256x256 с охватом 250 км от радара.
