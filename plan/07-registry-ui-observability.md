# Этап 7. Реестр моделей, графики и UI-наблюдаемость

## Проблема

График обучения сохраняется только после полного завершения процесса. При
прерывании модель остается в статусе `training`, UI показывает ее в реестре, а
график отсутствует.

## Цель

Сделать обучение наблюдаемым, восстанавливаемым и честно представленным в UI.

## Статусы модели

- `queued`
- `training`
- `completed`
- `failed`
- `interrupted`
- `rejected_quality_gate`
- `published`

## Задачи backend

1. Сохранять после каждой эпохи:
   - checkpoint;
   - `history.csv`;
   - `metrics.json`;
   - `learning_curve.png`;
   - metadata status.
2. Оборачивать обучение в `try/except/finally`.
3. При остановке процесса выставлять `interrupted`.
4. Сохранять traceback или краткую ошибку для `failed`.
5. Добавить endpoint статуса обучения и текущих метрик.
6. Отделить лучший checkpoint от последнего checkpoint.
7. Публиковать модель только после quality gate этапа 6.
8. Не включать незавершенные модели в список operational-моделей.

## Задачи UI

1. Показывать progress по эпохам.
2. Обновлять график во время обучения.
3. Показывать:
   - train/validation loss;
   - baseline metrics;
   - CSI по порогам;
   - метрики по lead time;
   - pipeline version;
   - dataset provenance;
   - QC status.
4. Отдельно показывать статус источника:
   - source;
   - station;
   - UTC timestamp;
   - возраст последнего кадра;
   - пропуски;
   - реальные данные или demo.
5. Запретить кнопку `Использовать` для непрошедших gate моделей.

## Тесты

- График существует после первой эпохи.
- Прерванное обучение получает статус `interrupted`.
- Исключение обучения получает статус `failed`.
- UI не предлагает `training` и `rejected_quality_gate` модель для инференса.
- API корректно отдает промежуточные метрики.

## Риски

- Текущий `TaskRunner` хранит состояние только в памяти Flask-процесса.
  Для первого этапа допустимо сохранить файловую модель состояния; позднее
  можно вынести задачи в отдельный job runner.

## Критерий завершения

Пользователь видит графики и статус во время обучения, а operational registry
содержит только завершенные и проверенные модели.
