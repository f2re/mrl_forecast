# Этап 6. Бейзлайны, метрики и новая модель

## Проблема

Текущая ConvLSTM оптимизирует только MSE, использует узкий последний hidden layer
и может выдавать физически бессмысленное однородное поле. Без baseline невозможно
понять, приносит ли нейросеть пользу.

## Цель

Построить измеримый ML-контур: сначала простые бейзлайны, затем улучшенная модель,
затем gate качества.

## Обязательные бейзлайны

1. **Persistence**
   - Последний наблюденный кадр повторяется на все lead times.
2. **Advection / optical flow**
   - Оценивается поле перемещения;
   - последний кадр переносится вперед по времени.
3. **Climatology / zero-rain**
   - Используется как диагностический нижний baseline для дисбаланса классов.

## Метрики

Считать отдельно для каждого lead time:

- MSE и MAE;
- CSI по порогам `5`, `10`, `20`, `30 dBZ`;
- POD;
- FAR;
- FSS на согласованных пространственных масштабах;
- bias по площади осадков;
- долю пикселей прогноза выше каждого порога;
- SSIM только как дополнительную визуальную метрику.

## Архитектура первого улучшения

Не переходить сразу к сложным генеративным моделям. Сначала:

1. Убрать bottleneck `[hidden_channels, 1]`.
2. Использовать полноценный encoder-decoder ConvLSTM.
3. Добавить ограниченный выход:
   - `sigmoid` для нормализованного диапазона;
   - либо другой явно документированный способ.
4. Добавить weighted loss или составной loss для редких осадков.
5. Добавить early stopping и model selection по набору метрик, а не только MSE.
6. Сохранять seed и конфигурацию запуска.

После стабильного baseline можно сравнить:

- TrajGRU;
- PredRNN;
- residual advection correction;
- probabilistic forecast;
- генеративные подходы для длинных lead times.

## Диагностика зеленого слоя

Добавить автоматический reject, если:

- поле почти константно;
- доля осадков резко отличается от входа и climatology;
- модель хуже persistence;
- значения выходят за допустимый диапазон;
- spatial variance ниже минимального порога.

## Тесты

- Persistence baseline работает на fixture.
- Метрики проверены на ручных маленьких массивах.
- Модель не выдает значения вне допустимого диапазона.
- Uniform forecast detector ловит текущий зеленый слой.
- Training smoke-тест сохраняет checkpoint и метрики.

## Gate допуска модели

Модель не публикуется в operational registry, если:

- не прошла frozen test;
- хуже persistence по согласованному набору метрик;
- не имеет QC report;
- не совместима с pipeline version;
- выдает uniform field anomaly.

## Критерий завершения

Новая модель проходит gate, а ее преимущество над persistence и advection
baseline подтверждено на независимом test-наборе.
