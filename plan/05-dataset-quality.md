# Этап 5. Качество датасета и честная валидация

## Проблема

Соседние последовательности почти идентичны: окно из 8 кадров сдвигается на один
кадр. Случайный split помещает связанные окна одновременно в train и validation.
Кроме того, один день одного радара недостаточен для устойчивой модели.

## Цель

Создать версионируемые датасеты с QC, независимыми split и понятным
метеорологическим покрытием.

## Задачи

1. Ввести manifest каждого кадра:
   - station;
   - UTC timestamp;
   - source file;
   - product;
   - pipeline version;
   - coverage;
   - QC flags.
2. Выполнять временной resampling до формирования окон.
3. Делить данные до нарезки окон:
   - по дням;
   - по событиям;
   - при необходимости по станциям.
4. Не допускать перекрытия кадров между train, validation и test.
5. Добавить QC-фильтры:
   - NaN и mask coverage;
   - слишком малая площадь валидных данных;
   - константные поля;
   - аномальные dBZ;
   - пропуски времени;
   - decode errors;
   - demo provenance.
6. Формировать отчет распределений:
   - доля сухих кадров;
   - доля пикселей выше `5`, `10`, `20`, `30 dBZ`;
   - распределение по станциям, месяцам, часам и событиям.
7. Разделить датасеты по назначению:
   - `development-small`;
   - `training`;
   - `validation`;
   - `test-frozen`.
8. Хранить dataset manifest и QC report рядом с массивами.
9. Рассмотреть Zarr для крупных наборов после стабилизации схемы.

## Минимальный набор данных

Для инженерной проверки достаточно нескольких независимых дней. Для оценки
качества модели нужен набор из нескольких месяцев, содержащий:

- сухую погоду;
- слабые осадки;
- конвективные события;
- фронтальные осадки;
- разные сезоны;
- разные станции.

## Тесты

- Ни один source timestamp не встречается в нескольких split.
- Demo-кадры исключены.
- Окно с временным разрывом отклоняется или маркируется согласно политике.
- Manifest соответствует фактическим `.npy` или Zarr данным.
- QC report воспроизводим.

## Риски

- После честного split метрики могут заметно ухудшиться. Это исправление оценки,
  а не регрессия качества.
- Данные NOAA NEXRAD и российских ДМРЛ нельзя смешивать до унификации продукта и
  QC.

## Критерий завершения

Каждый датасет имеет manifest, QC report, pipeline version и независимые split
без пересечения исходных кадров.
