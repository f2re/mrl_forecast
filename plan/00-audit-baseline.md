# Этап 0. Аудит и воспроизводимый baseline

## Назначение

Зафиксировать текущее поведение до изменений. Этот этап не улучшает систему, но
создает эталон для проверки каждого последующего исправления.

## Фактические результаты аудита

### AWS

- Публичный архив NOAA NEXRAD Level II доступен через бакет
  `unidata-nexrad-level2`.
- Read-only проверка вернула:
  - `KOKX`, `2024-05-20`: `192` скана;
  - `KOKX`, `2026-05-13`: `227` сканов.
- Локально уже существует архив
  `data/raw/archive/kokx_2026-05-13_082925`.
- При неявном регионе boto3 live-запрос пытался использовать
  `unidata-nexrad-level2.s3.ru-central1.amazonaws.com` и падал.

### Временная шкала

- Для локального архива KOKX найдено `178` пригодных файлов.
- Медианный интервал между сканами: `8.62` минуты.
- Минимальный интервал: `5.18` минуты.
- Максимальный интервал: `9.88` минуты.
- Текущая логика 15-минутных шагов не соответствует исходным данным.

### Датасет

- Подготовленный датасет:
  `data/processed_archive/dataset_kokx_20260530_084913`.
- Количество последовательностей: `171`.
- Длина последовательности: `8`.
- Форма кадра: `256 x 256`.
- Средняя доля нулей: около `89%`.
- Соседние последовательности перекрываются на `7/8` кадров.

### Модель

- Проверенная модель:
  `models/registry/model_20260530_091154/best_model.pt`.
- Для контрольного примера прогноз первого шага покрывает `100%` пикселей
  положительной отражаемостью.
- Среднее поле первого шага: около `28 dBZ`.
- На четвертом шаге поле остается почти однородным: около `24 dBZ`.
- На контрольном примере MSE модели примерно в 40 раз хуже persistence.
- В метаданных модель остается в статусе `training`.
- `learning_curve.png` отсутствует.

### Карта

- В `src/map_visualization.py` используется `origin='upper'`.
- Гриддинг строит ось `y` от отрицательных значений к положительным.
- Следствие: кадр отображается зеркально по оси север-юг.
- Смещения в метрах локальной сетки напрямую добавляются к Web Mercator.
  Для широты KOKX масштаб Web Mercator искажен примерно в `1.32` раза.

## Задачи

1. Создать каталог контрольных артефактов, исключенный из production registry.
2. Зафиксировать один архивный файл KOKX и его метаданные.
3. Зафиксировать статистику исходного радара и подготовленной сетки:
   `shape`, `dtype`, `min`, `max`, `mean`, `NaN`, mask coverage.
4. Сохранить текущий рендер карты как отрицательный эталон.
5. Сохранить текущий прогноз зеленого слоя как отрицательный эталон.
6. Добавить скрипт аудита, который повторяет эти проверки одной командой.

## Артефакты этапа

- `tests/fixtures/` с минимальными контрольными файлами.
- `scripts/audit_baseline.py`.
- `docs/audit-baseline.md` или машинно-читаемый JSON-отчет.
- Изображения before/after для географии и прогноза.

## Проверки

- Аудит запускается без обучения модели.
- Результаты содержат версии библиотек.
- Результаты содержат абсолютные UTC timestamp исходных данных.
- Контрольный отчет явно помечает текущую модель как не прошедшую baseline gate.

## Критерий завершения

Одна команда воспроизводит текущие дефекты и создает отчет, пригодный для
сравнения после каждого этапа.
