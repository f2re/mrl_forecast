# Этап 4. Надежный AWS ingestion

## Проблема

AWS-источник NOAA NEXRAD работает, но код зависит от локальной настройки boto3.
Архивный загрузчик и live inference конфигурируют регион по-разному.

## Цель

Сделать AWS ingestion воспроизводимым, наблюдаемым и независимым от локального
AWS-профиля пользователя.

## Зафиксированный источник

- Dataset: NOAA NEXRAD Level II.
- Публичный бакет: `unidata-nexrad-level2`.
- Регион: `us-east-1`.
- Доступ: unsigned public S3.
- Текущий Python-клиент: `nexradaws`.

## Задачи

1. Вынести создание AWS-клиента в одну фабрику.
2. Явно задавать `region_name='us-east-1'`.
3. Явно включить unsigned S3 access.
4. Удалить зависимость от случайных `AWS_REGION` пользователя.
5. Фильтровать `_MDM` до формирования последовательности.
6. Парсить timestamp каждого скана и сортировать по времени.
7. Заменить жесткий поиск 15-минутных интервалов на resampling из этапа 2.
8. Добавить retry с ограничением числа попыток и понятными ошибками.
9. Добавить API health-check:
   - доступность списка сканов;
   - время последнего скана;
   - возраст данных;
   - загрузка одного файла;
   - успешное декодирование;
   - статистика grid.
10. Сохранять source metadata в raw archive.

## Проверки

- Запуск с локальным регионом `ru-central1` все равно обращается к `us-east-1`.
- Read-only запрос KOKX за `2024-05-20` возвращает непустой список.
- `_MDM` исключаются из последовательностей.
- Один файл загружается и проходит единый preprocessing pipeline.
- Ошибка сети возвращается как `source_failed`.

## Интеграционный smoke-тест

```bash
AWS_EC2_METADATA_DISABLED=true \
AWS_DEFAULT_REGION=us-east-1 \
AWS_REGION=us-east-1 \
venv/bin/python scripts/check_aws_source.py --station KOKX --date 2024-05-20
```

## Риски

- `nexradaws` является внешней зависимостью без жесткой фиксации версии.
- Долгосрочно лучше предусмотреть собственный тонкий S3-клиент либо контрактный
  wrapper, чтобы смена библиотеки не затронула остальную систему.

## Критерий завершения

AWS smoke-тест стабильно получает, скачивает и декодирует контрольный файл вне
зависимости от локального AWS-профиля.
