### 1. Официальные источники РФ и Глобальные архивы

Сырые данные (HDF5/BUFR) для глубокого обучения лежат здесь, но требуют запросов.

1. **ВНИИГМИ-МЦД (ФГИС ЕГФД)** (`fund.meteo.ru`) — Единственный официальный государственный архив РФ. Выдает исторические сырые данные ДМРЛ (полярные сканы) и МРЛ. Для университетов и НИИ по официальному письму данные предоставляются бесплатно.
2. **Сеть ДМРЛ Росгидромета** (`orad.meteorf.ru`) — Официальный портал оперативных данных. Сами сырые архивы там не лежат, но сервис отдает JSON-метаданные и тайлы, которые можно парсить для сбора текущего датасета.
3. **Гидрометцентр РФ (Радары)** (`meteoinfo.ru/radar`) — Агрегатор карт метеолокаторов. Источник для написания парсеров, собирающих PNG-отражаемость для обучения сверточных нейросетей (CNN).
4. **WMO OSCAR** (`oscar.wmo.int`) — Каталог ВМО. Содержит точные координаты, частоты, углы возвышения и технические характеристики (метаданные) всех российских МРЛ/ДМРЛ. Необходимо для калибровки моделей ИИ.
5. **EUMETSAT ODYSSEY** (`data.eumetsat.int`) — В европейские композиты часто попадают данные с приграничных российских радаров (Калининград, Санкт-Петербург, Мурманск). Доступен HDF5 архив.

### 2. Агрегаторы с открытым API (с покрытием России)

Если для ИИ достаточно 2D-карт (отражаемости) или композитов в реальном времени.
6. **RainViewer API** (`[rainviewer.com/api.html](https://rainviewer.com/api.html)`) — Крупнейший агрегатор радаров, собирающий данные в том числе с Росгидромета. Отдает сырые значения (dBZ) и готовые маски осадков. Идеально для задач Nowcasting.
7. **RainViewer GitHub** (`[github.com/rainviewer](https://github.com/rainviewer)`) — Репозитории разработчиков платформы. Там лежат скрипты интеграции и примеры работы с их API на Python.
8. **Yandex Weather API** (`yandex.ru/dev/weather/`) — Использует данные сети ДМРЛ для построения карт осадков. Для коммерческих/исследовательских проектов можно получить доступ к историческим срезам.
9. **OpenWeatherMap Radar API** (`openweathermap.org/api/radar`) — Глобальные радарные композиты, включая территорию РФ, с исторической глубиной для Machine Learning.

### 3. Библиотеки Python для работы с сырыми данными ДМРЛ (GitHub)

Российские ДМРЛ-С экспортируют данные в форматах **BUFR**, **HDF5 (ODIM_H5)** и проприетарных **.vol** файлах. Для их тензоризации нужны эти библиотеки:
10. **wradlib** (`[github.com/wradlib/wradlib](https://github.com/wradlib/wradlib)`) — Инструмент №1. Поддерживает ODIM_H5 (современный стандарт Росгидромета). В issues на GitHub можно найти патчи от энтузиастов для чтения старых российских `.vol` файлов. Умеет фильтровать помехи и переводить полярные координаты в декартовы массивы NumPy.
11. **Py-ART** (`[github.com/ARM-DOE/pyart](https://github.com/ARM-DOE/pyart)`) — Мощная библиотека от Минэнерго США. Отлично работает с HDF5 и NetCDF, идеальна для подготовки 3D-тензоров для обучения 3D-CNN (сверточных сетей).
12. **ecCodes** (`[github.com/ecmwf/eccodes](https://github.com/ecmwf/eccodes)`) — Библиотека на C/Fortran от ECMWF. Стандарт для декодирования бинарных `.bin` файлов BUFR (формат старых радаров МРЛ-5).
13. **eccodes-python** (`[github.com/ecmwf/eccodes-python](https://github.com/ecmwf/eccodes-python)`) — Python-обертка для п.12. Перегоняет BUFR-таблицы в массивы Pandas/NumPy.
14. **h5py** (`[github.com/h5py/h5py](https://github.com/h5py/h5py)`) — Низкоуровневая библиотека для распаковки `.h5` файлов, если вам нужны только сырые массивы (dataset) и метаданные без географической привязки (например, для обучения трансформеров на временных рядах).

### 4. Утилиты, конвертеры и скраперы (GitHub)

Проекты, которые помогут выкачать, конвертировать и визуализировать данные.
15. **LROSE-core** (`[github.com/NCAR/lrose-core](https://github.com/NCAR/lrose-core)`) — Базовый C++ софт от NCAR (США). Утилита `Radx` из этого пакета — это «швейцарский нож». Она способна конвертировать почти любой сырой проприетарный формат радара в стандартизированный `CF/Radial` (NetCDF), который легко "съест" любой ИИ-фреймворк (PyTorch/TensorFlow).
16. **rack** (`[github.com/fmidev/rack](https://github.com/fmidev/rack)`) — C++ программа от Финского метеорологического института (FMI). Заточена под обработку ODIM HDF5. Умеет аппаратно вычищать неметеорологические шумы (птицы, здания) перед подачей данных в нейросеть.
17. **Radar-Visualisation** (`[github.com/martinschilliger/Radar-Visualisation](https://github.com/martinschilliger/Radar-Visualisation)`) — Bash-скрипты и Docker-контейнеры для нарезки сырых HDF5/NetCDF файлов в XYZ-тайлы. Полезно для визуализации результатов предсказаний ИИ на веб-картах.
18. **OpenStorm** (`[github.com/JordanSchlick/OpenStorm](https://github.com/JordanSchlick/OpenStorm)`) — Open-source 3D просмотрщик радарных объемов. Поддерживает ODIM H5. Полезен для ручного визуального анализа обучающей выборки (поиска аномалий в 3D).
19. **mlcast-dataset-DMI-radar_precipitation** (`[github.com/mlcast-community/mlcast-dataset-DMI-radar_precipitation](https://github.com/mlcast-community/mlcast-dataset-DMI-radar_precipitation)`) — Отличный пример пайплайна (на Python) для конвертации сырых радаров HDF5 в формат Zarr, который является стандартом де-факто для обучения ИИ на огромных погодных датасетах.
20. **Самописные парсеры (Поиск по GitHub):** По запросам `github.com search "meteoinfo parser"`, `"dmrl parser"`, `"Росгидромет API"` можно найти скрипты на Python от российских разработчиков, которые ставятся на `cron` и по расписанию скачивают текущие снимки радаров ДМРЛ с открытых сайтов, формируя локальный датасет.
